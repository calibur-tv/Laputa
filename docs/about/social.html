# 前言：

  1. 任何一个互联网产品，将其功能属性进行归类，都可以分成：社交属性和工具属性两种
    • QQ、微信、微博 的主要属性是社交（连接人与人之间的媒介），次要属性是工具
    • 美颜相机、支付宝、优酷、爱奇艺...基本只有工具属性，没有社交价值
  2. 一个只有工具属性的产品，盈利途径是贫乏的，如：
    • 【向社交转型】：超级课程表作为一个工具软件，为了盈利，强行加入社交元素，导致产品死掉
    • 【接商业广告】：如一些文件管理器、垃圾清理工具...虽然能够盈利，但因为广告破坏用户体验，用户会慢慢流失
    • 【靠技术壁垒】：依靠技术壁垒盈利的产品，最终只会被大公司（腾讯、阿里、小米）低价收购，或被大公司的入局后的价格战淘汰，除非你足够有钱（抖音、快手）
    • 【靠资源壁垒】：如优酷、爱奇艺、腾讯动漫这种产品，只能靠版权资源来维持，需要大量的资本投入以及一个好的推广平台（优酷被阿里控股，爱奇艺被百度控股，腾讯动漫就是腾讯的产品）
    • 因此：一个没有技术壁垒，没有资本背景的工具属性的产品，如果没有发展成社交的可能（超级手电筒、美颜相机）那就只能植入广告来快速变现，如果有发展成社交的可能（超级课程表）那么最长远的发展方向就是向社交转型
  3. 一个只有社交属性的产品，在中国已经没有很好的机会了：

    • 社交的入口，只有：QQ、微信

    • 想要在社交上突破，要么是垂直社交，要么是基于场景的社交（或者可以说是基于工具基础的社交）

      垂直社交：由于QQ、微信等产品面向的用户量太庞大，所以无法精确的为每一个小群体来提供个性化功能，因此专门为某个小群体提供个性化功能的产品就可以发展起来，如： - 一大堆的相亲平台，兴趣社交平台 都是这样的，他们的盈利模式也只有：卖会员、送礼物、接商业广告 - 这样的产品，如果没有工具属性作为支撑，现阶段基本不可能做大的

      工具社交：具有一些个性化功能的社交产品，如： - 微博：基于新闻媒体的社交，当年战胜了腾讯微博后活了下来 - 知乎：基于专业问答的社交，知乎能发展起来，完全是因为百度知道和腾讯问问在竞争，但这两个都不是公司的主营产品，所以没有认真发展

  4. 一个兼具社交属性与工具属性的产品，能够发展壮大：
    • 如上面的知乎、微博都是在一定的工具基础上发展出来的社交，然后慢慢壮大
    • 但也有从工具转型成社交失败的例子，比如超级课程表，比如之前的支付宝
  5. 总结一下就是：工具属性和社交属性如何找方向，未来的发展是什么，说的再多也都是一样的逻辑。

# 我们这样做的好处是什么？

一个产品，包含了多个竞品的功能，就像一个大杂烩，这样做有什么好处吗？

  1. 我个人的时间精力、技术实力、运营资本都是无法与竞品相比的，因此我们不会试图在单一功能上超越竞品
  2. 各个竞品在单一功能上比我们强，用户面也更大，但也由于竞品面向的用户群体太笼统了，导致无法为某一特殊用户群体做垂直化研发，比如：
    • 百度贴吧的用户量极大，而动漫爱好者仅仅是其中的一小部分，所以贴吧没有看动漫和看漫画的功能，它只具备社交属性没有工具属性
    • 豆瓣电影的评分功能是针对电影而设立的，完全不适用于来给动漫打分，专业的动漫评分平台在国内其实是一个缺失
    • 知乎的问答氛围比较好，但是主要用户年龄段在20岁以上，这些用户已经过了感性的去看动漫的年龄，因此对那些在孩童时期（15岁左右）喜欢的作品无法感同身受，进行友好的讨论
  3. 工具属性上，我们无法胜过bilibili，事实上现阶段没有哪家视频为主的公司能胜过bilibili，因此我们只能在社交属性上寻求突破
  4. 社交属性上，国内还没有一个氛围良好的为动漫爱好者垂直开发的社交平台
    • acfun因为自身发展问题，基本已经退出了历史舞台
    • bilibili是一个强工具属性的产品，并且也在努力向社交转型
    • 第一弹是靠着色情内容起家的产品，目前已经通过游戏运营实现了盈利，社区靠着极个别高质量的用户撑着，该产品涉及数据造假，因此称不上一个强劲的竞争对手
    • 其它竞品如半次元等并没有很强的社交属性
  5. 因此，我们的逻辑是：
    • 前期通过看动漫、看漫画的工具属性吸引种子用户（已实现这个阶段）
    • 通过开发各种社交功能，为用户提供一个良好的社区氛围来吸引用户（已实现产品的自我造血功能）
    • 当用户量积累到一定程度，用户忠诚度保持高水准之后，开始通过附加功能盈利（或开发／代理游戏）

# 为什么竞品不做我们做的事？

  1. 具有社交属性的竞品：百度贴吧、豆瓣、知乎，他们的用户群体更加庞大，而非某一特殊的用户群体，因此他们不会针对部分用户来开发特殊功能（目前能够做到全面，又能做到细分的，只有微信+小程序，但是我们的目标用户年龄段更低，喜欢用QQ，很少用微信）
  2. 具有工具属性的竞品：bilibili 在向着社交方向发展，但并不顺利，详细的报告会在“竞品分析”报告中给出

我们做了好几个竞品的功能，是否意味着我们的成本也会是竞品的许多倍？其实一个产品的生命周期中，研发成本并不会占非常大的份额，最花钱的还是推广（今日头条App开屏广告，1天的价格在150万上下）

# 这样的模式，是否有成功的先例 ？

  1. 从功能上来讲，并没有我们这样的融合了多家社交平台核心功能的产品出现过，因此并没有成功的先例
  2. 但是从产品思路上来讲，是有成功先例的，那就是 bilibili
    • bilibili 是一个视频弹幕网站，早年因为版权的管理还不严格，所以靠着极强工具属性发展了起来，到目前为止，它的工具属性也是无法被国内其他视频网站超越的
    • 而弹幕，是一种如同“烟花”一样的，可以烘托氛围的功能，对于一个社区而言，氛围是极其重要的，因此 bilibili 能在早年战胜 acfun 很大一部分原因是 bilibili 的弹幕体验比 acfun 好
    • 因此 bilibili 的发展模式就是：以强工具属性作为基础 -> 靠着极强的社区氛围积累用户量，提高用户忠实度 -> 用户量足够大之后，开始通过小功能（广告等）来盈利 -> 向着社交转型，扩大盈利途径 -> 上市 -> ...
    • 我们的发展模式就是：以强大的社交属性作为基础 -> 靠着极强的社区氛围积累用户量，提高用户忠实度 -> 用户量足够大之后，开始通过小功能（广告等）来盈利 -> 向着工具转型，扩大行业影响力 -> 上市 -> ...
  3. 因此我们走的路和 bilibili 相同，只不过因为大环境的原因，选择了另一个切入点

# 为什么是这几个功能，而不是其它功能，这些功能是不是伪需求？

目前的动漫爱好者日常情况是这样的：

  1. 找到一部自己喜欢的动漫、漫画开始看
  2. 看到自己喜欢的剧情之后，就会产生分享的欲望，并且这个分享的欲望大部分是希望得到反馈（共鸣）的
  3. 因此 bilibili 的弹幕就是一个很好的表达途径，然而随着弹幕质量越来越低，更多的人看视频时不再喜欢开启弹幕，弹幕营造的良好氛围在慢慢削弱甚至在破坏气氛
  4. 其它途径就是视频板块下的评论区，然而无论是弹幕还是评论区，参与的人都是极少的（因为没有人看），并且属于陌生人社交（发弹幕的概率是万分之一，发评论的概率是千分之一）（详细的统计数据会在竞品分析报告中给出）
  5. 如果用户想与自己熟悉的人分享，最普遍的有两种途径：
    • 在QQ群里与大家分享，然而QQ群由于消息流的关系，自己发表的内容很快就会被冲走，不会有很多人注意到
    • 发说说来分享，这是一种比较好的途径，然而一旦说说的质量太低，反而会影响一个用户的个人形象
  6. 如果用户想发表一些详细的观点，弹幕和评论区这种轻量的表达途径都是不适合的，而说说的社交又是在自己的熟人圈里，并非是兴趣圈，因此这个时候的表达途径一般有两个：
    • 百度贴吧，贴吧偶尔会出现一些非常精彩的分析帖子，在前些年非常火，然而随着贴吧氛围越来越差，高质量的内容也越来越少
    • 知乎，知乎目前在某些动漫如《海贼王》的讨论非常激烈，基本每周的漫画都能上热搜榜
  7. 我们做的这几个板块就可以很好的处理好这一社交生命周期： - 一部动漫/漫画即将上映，我们通过帖子板块（置顶且打上新闻标签）来告知用户 ，起到资讯的工具的作用 - 上映后，我们若有资源，就可以在站内提供资源给用户观看，起到看资源的工具作用 - 用户看完一集后，可以发评论来表达想法，起到社交属性的作用 - 用户有自己对剧情的个人见解后，可以发帖（打上不同的分类标签），来表达自己的思想 - 动漫/漫画有一些值得深入讨论的剧情出现后，可以提问题，然后等待漫友的回答，工具/社交属性 - 内容完结后，可以通过漫评功能对整个剧情进行整体分析，给作品打分，起到工具的作用

总结一下目前的情况就是：用户会在版权方的平台观看内容，然后去其它平台发表观点，比较有名的就是：“我在鼠绘漫画看海贼王，然后在知乎提问”的情况，因此这种二次元社区的需求，并非伪需求，而是一个存在着的，一直没能够很好建立起来的社交体系。

# 如何管理这么多的功能？

社区的管理，主要是维持社区的氛围，虽然我们的功能多，但是已经将用户进行了分层，是有利于社区管理的

  1. 动漫爱好者的年龄段跨度极大，从 10 ~ 25 岁不等，这些用户的价值观、性格多种多样，如果将这些用户集中在一个池子里，必然会引发矛盾，然而动漫用户中尚未形成健全价值观的不在少数，从而引发群体间更大的冲突，就像现在 bilibili 的弹幕区，基本已经沦陷了。
  2. 我们通过开发出不同类型的社区板块，将不同表达习惯的用户划分到不同的功能区
    • 评论区：作为快餐区，用来让用户表达一时的想法就好
    • 帖子区：作为新闻资讯的工具和作为大部分用户表达想法的工具，这类想法一般是缺乏逻辑严谨性的猜想等，但是这些内容作为未成年人思想的结晶也非常有价值
    • 图片区：是专门用来发表手绘、精美图片、cosplay等板块，这个板块对于ACG领域来说，是一个及其重要的表达途径
    • 问答区：承载着每一话的分析，包括剧情分析，人物分析等，内容严谨性较帖子区高
    • 漫评区：承载着一部作品的精华分析，表达门槛最高，更容易产出高质量内容，这里的内容是少而精，帖子区的内容是多而杂
  3. 因此，我们只需要在用户表达途径的选择上加强引导，社区的管理就会轻松很多了

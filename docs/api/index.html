# calibur.tv 接口文档

# 全局配置
  1. 接口的host在 staging 环境是：https://t-api.calibur.tv，production 下是：https://api.calibur.tv
  2. 用户认证使用 JWT-Auth，需要用户认证的接口必须携带指定的 header 字段，例：
    requestHeader = {
  'Authorization': 'Bearer ' + 'userTokenString'
}
  3. 移动端在发请求的时候，要额外携带以下参数做校验：
    requestHeader = {
  'Accept': 'application/x.api.latest+json'
}
    Accept中的latest要替换为当前 API 的版本号，如 v1，v2...
  4. 接口的返回统一如下：
    response = {
 code: responseCode('type:integer'),
 data: responseData('type:any')
}
    无论请求成功或失败都会返回上述结构的数据，特别说明：
    1. 请求成功时，code 是 0，其它失败时，是 http status code
    2. response 的 status code 在 code === 0 时是 200，其它时候与 response.code 一致
    3. data 在返回成功时，会是任意可能的类型，返回失败时，是错误信息字符串
  5. 接口超时设置为10s，如果超时则显示预定义的错误
